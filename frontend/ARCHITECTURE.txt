┌──────────────────────── DATA FLOW ─────────────────────────────────────┐
│                                                                         │
│  1. User Action (Browser)                                              │
│     → Click "Onboard Client" button                                    │
│                                                                         │
│  2. API Request (Frontend → Backend)                                   │
│     → POST /api/clients/onboard                                        │
│     → Body: { name, slug, website, crawl_site }                        │
│                                                                         │
│  3. Command Execution (Backend → CLI)                                  │
│     → subprocess.run([python, agentctl.py, ...])                       │
│     → Creates directories, scaffolds files                             │
│                                                                         │
│  4. WebSocket Broadcast (Backend → All Clients)                        │
│     → { type: "client_onboarded", slug: "demo-hvac" }                  │
│                                                                         │
│  5. UI Update (Frontend)                                               │
│     → Toast notification appears                                       │
│     → React Query refetches client list                                │
│     → Dashboard updates with new stats                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
